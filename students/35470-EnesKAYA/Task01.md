# Event-Driven Air Defense System Architecture

## Overview

This document describes a simplified **Event-Driven Architecture (EDA)** for an air defense system similar to systems like Iron Dome.

The system reacts to **events** generated by sensors and other subsystems. Each component processes events and may emit new events that trigger further actions.

---

# Architecture

```
Radar → Event Bus → Processing Services → Command System → Launcher
```

Main idea:

- Sensors produce events
- Event bus distributes them
- Services react to events
- Defensive actions are triggered

---

# Core Components

## Radar System

Detects airborne objects and emits detection events.

**Produces:**

- `RadarDetectionEvent`

Example:

```json
{
  "eventType": "RadarDetectionEvent",
  "objectId": "OBJ-123",
  "speed": 950,
  "altitude": 12000,
  "heading": 210,
  "timestamp": "2026-03-05T14:03:22Z"
}
```

---

## Event Bus

Central communication layer.

Responsibilities:

- Distribute events to subscribers
- Decouple producers and consumers
- Allow asynchronous processing

Example technologies:

- Apache Kafka
- RabbitMQ
- AWS EventBridge

---

# Event Flow

Typical sequence of events in the system.

```
RadarDetectionEvent
      ↓
ObjectTrackedEvent
      ↓
ObjectClassifiedEvent
      ↓
ThreatConfirmedEvent
      ↓
InterceptDecisionEvent
      ↓
InterceptorLaunchedEvent
      ↓
InterceptSuccessEvent
```

---

# Event Types

| Event | Description |
|------|-------------|
| RadarDetectionEvent | Radar detects object |
| ObjectTrackedEvent | Tracking service updates trajectory |
| ObjectClassifiedEvent | Object classified (drone, missile, aircraft) |
| ThreatConfirmedEvent | Object determined to be hostile |
| InterceptDecisionEvent | Command system decides to intercept |
| InterceptorLaunchedEvent | Interceptor missile launched |
| InterceptSuccessEvent | Target destroyed |
| InterceptFailedEvent | Intercept unsuccessful |

# Event Handlers

Each subsystem reacts to specific events.

## Tracking Service

**Consumes:**

- `RadarDetectionEvent`

**Produces:**

- `ObjectTrackedEvent`

---

## Classification Service

**Consumes:**

- `ObjectTrackedEvent`

**Produces:**

- `ObjectClassifiedEvent`

---

## Threat Assessment Service

**Consumes:**

- `ObjectClassifiedEvent`

**Produces:**

- `ThreatConfirmedEvent`

---

## Command System

**Consumes:**

- `ThreatConfirmedEvent`

**Produces:**

- `InterceptDecisionEvent`

---

## Launcher Control

**Consumes:**

- `InterceptDecisionEvent`

**Produces:**

- `InterceptorLaunchedEvent`

---

# Example Processing Logic

Example event handler:

```pseudo
on ThreatConfirmedEvent:

  if predictedImpactZone == protectedArea:
      emit InterceptDecisionEvent
```

---

# Advantages of Event-Driven Architecture

- Loose coupling between components
- High scalability
- Real-time reaction to threats
- Fault isolation
- Parallel processing
